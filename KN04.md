# KN04: Docker Compose

## A) Docker Compose: Lokal

### Teil a) Verwendung von Original Images

#### Ordnerstruktur

```
kn04/
├── docker-compose.yml
├── Dockerfile
├── info.php
└── db.php
```

---

#### db.php (angepasst)

```php
<?php
$db_server = "m347-kn04a-db";
$db_username = "root";
$db_password = "rootpassword";
$db_name     = "mysql";

$connection = mysqli_connect($db_server, $db_username, $db_password, $db_name);

if (!$connection) {
    die("Verbindung fehlgeschlagen: " . mysqli_connect_error());
}

echo "Verbindung erfolgreich!";
mysqli_close($connection);
?>
```

> Der Hostname entspricht dem Container-Namen des DB-Containers.

---

#### Dockerfile (aus KN02)

```dockerfile
FROM php:8.0-apache
WORKDIR /var/www/html
COPY info.php .
COPY db.php .
RUN docker-php-ext-install mysqli
EXPOSE 80
```

---

#### docker-compose.yml (Teil a)

```yaml
networks:
  kn04-network:
    driver: bridge
    ipam:
      config:
        - subnet: 172.10.0.0/16
          ip_range: 172.10.5.0/24
          gateway: 172.10.5.254

services:
  db:
    image: mariadb:latest
    container_name: m347-kn04a-db
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: mysql
      MYSQL_USER: dbuser
      MYSQL_PASSWORD: dbpassword
    networks:
      - kn04-network

  web:
    build: .
    container_name: m347-kn04a-web
    ports:
      - "80:80"
    depends_on:
      - db
    networks:
      - kn04-network
```

---

#### Was macht `docker compose up`?

| Schritt | Befehl (intern) | Erklärung |
|---|---|---|
| 1 | `docker network create` | Erstellt das definierte Netzwerk (`kn04-network`) |
| 2 | `docker build` | Baut das Image für den Webserver anhand des Dockerfiles |
| 3 | `docker pull` | Zieht das `mariadb:latest` Image von Docker Hub |
| 4 | `docker create` | Erstellt die Container mit den definierten Einstellungen |
| 5 | `docker start` | Startet die Container in der richtigen Reihenfolge (wegen `depends_on`) |

Mit dem Flag `-d` (detached) läuft alles im Hintergrund.

### Teil b) Verwendung eigener Images

Das Dockerfile wird nicht mehr benötigt. Stattdessen wird das eigene publizierte Image von Docker Hub verwendet. Der IP-Range wird geändert.

#### docker-compose.yml (Teil b)

```yaml
networks:
  kn04-network:
    driver: bridge
    ipam:
      config:
        - subnet: 172.20.0.0/16
          ip_range: 172.20.5.0/24
          gateway: 172.20.5.254

services:
  db:
    image: mariadb:latest
    container_name: m347-kn04b-db
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: mysql
      MYSQL_USER: dbuser
      MYSQL_PASSWORD: dbpassword
    networks:
      - kn04-network

  web:
    image: DEIN-DOCKERHUB-USERNAME/DEIN-IMAGE:latest
    container_name: m347-kn04b-web
    ports:
      - "80:80"
    depends_on:
      - db
    networks:
      - kn04-network
```

> `DEIN-DOCKERHUB-USERNAME/DEIN-IMAGE:latest` durch den echten Image-Namen ersetzen.

---

#### Warum gibt db.php einen Fehler?

Das eigene Image wurde in KN02 mit dem Hostname `kn02b-db` gebaut. Dieser Hostname existiert im neuen Netzwerk nicht — der DB-Container heisst jetzt `m347-kn04b-db`.

Das Image weiss nichts vom neuen Netzwerk, weil es bereits mit den alten Verbindungsdaten gebaut wurde.

**Lösung:** Die `db.php` mit dem korrekten Hostnamen anpassen und das Image neu bauen und auf Docker Hub pushen.

## B) Docker Compose: Cloud

### Cloud-Init Datei

```yaml
#cloud-config

users:
  - name: ubuntu
    ssh_authorized_keys:
      - ssh-rsa DEIN_PUBLIC_KEY
      - ssh-rsa PUBLIC_KEY_LEHRPERSON

packages:
  - docker.io
  - docker-compose-plugin

write_files:
  - path: /home/ubuntu/docker-compose.yml
    content: |
      networks:
        kn04-network:
          driver: bridge
          ipam:
            config:
              - subnet: 172.10.0.0/16
                ip_range: 172.10.5.0/24
                gateway: 172.10.5.254

      services:
        db:
          image: mariadb:latest
          container_name: m347-kn04a-db
          environment:
            MYSQL_ROOT_PASSWORD: rootpassword
            MYSQL_DATABASE: mysql
            MYSQL_USER: dbuser
            MYSQL_PASSWORD: dbpassword
          networks:
            - kn04-network

        web:
          image: DEIN-DOCKERHUB-USERNAME/DEIN-IMAGE:latest
          container_name: m347-kn04a-web
          ports:
            - "80:80"
          depends_on:
            - db
          networks:
            - kn04-network

runcmd:
  - systemctl enable docker
  - systemctl start docker
  - cd /home/ubuntu && docker compose up -d

final_message: "KN04 Setup abgeschlossen!"
```
