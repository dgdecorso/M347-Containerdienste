# KN05: Datenspeicher mit Docker und Docker Compose

## A) Bind Mounts

In diesem Teil wird gezeigt, dass ein Ordner vom Host-System direkt mit einem Docker-Container geteilt werden kann.
Das simuliert den Fall, dass Code auf dem Host geändert wird und die Änderung sofort im Container sichtbar ist.

### Ordnerstruktur auf dem Host

```text
kn05/
└── mein_skript.sh
```

---

### `mein_skript.sh`

```bash
#!/bin/bash
echo "=========================================="
echo "Hallo aus dem Container kn05!"
echo "Dieses Skript ist ein Unikat von Diego."
echo "=========================================="
```

Nach der ersten Ausführung wurde die Datei auf dem Host geändert:

```bash
#!/bin/bash
echo "=========================================="
echo "Hallo aus dem Container kn05!"
echo "Dieses Skript ist ein Unikat von Diego."
echo "UPDATE: Diese Zeile wurde auf dem Host geändert."
echo "=========================================="
```

---

### Verwendete Befehle

| Schritt | Befehl | Erklärung |
|---|---|---|
| 1 | `docker run -d --name kn05 -v "${PWD}:/app" nginx` | Startet den Container `kn05` und bindet den aktuellen Host-Ordner nach `/app` ein. |
| 2 | `docker exec -it kn05 bash` | Öffnet eine Bash-Konsole im laufenden Container. |
| 3 | `cd /app` | Wechselt in den gemounteten Ordner. |
| 4 | `bash mein_skript.sh` | Führt das Skript im Container aus. |
| 5 | Datei auf dem Host ändern und speichern | Die Datei wird ausserhalb des Containers angepasst. |
| 6 | `bash mein_skript.sh` | Das Skript wird erneut ausgeführt und zeigt die Änderung sofort an. |

---

### Befehlsliste für Teil A

```bash
docker run -d --name kn05 -v "${PWD}:/app" nginx
docker exec -it kn05 bash
cd /app
bash mein_skript.sh
bash mein_skript.sh
```

## B) Volumes / Named Volumes

In diesem Teil wird gezeigt, dass zwei verschiedene Container dasselbe Named Volume verwenden können.
Beide Container schreiben in dieselbe Datei und können die Inhalte gegenseitig lesen.

Ein Named Volume wird von Docker verwaltet. Mehrere Container können dasselbe Volume verwenden und dadurch Daten teilen.

---

### Verwendete Befehle

| Schritt | Befehl | Erklärung |
|---|---|---|
| 1 | `docker volume create kn05-shared-volume` | Erstellt ein Named Volume. |
| 2 | `docker run -d --name kn05-volume-1 -v kn05-shared-volume:/daten nginx` | Startet den ersten Container mit dem Volume unter `/daten`. |
| 3 | `docker run -d --name kn05-volume-2 -v kn05-shared-volume:/daten nginx` | Startet den zweiten Container mit demselben Volume unter `/daten`. |

---

### Befehlsliste zum Starten der Container

```bash
docker volume create kn05-shared-volume
docker run -d --name kn05-volume-1 -v kn05-shared-volume:/daten nginx
docker run -d --name kn05-volume-2 -v kn05-shared-volume:/daten nginx
```

---

### Test: In Container 1 schreiben

```bash
docker exec -it kn05-volume-1 bash
cd /daten
echo "Nachricht von Container 1" >> austausch.txt
cat austausch.txt
exit
```

---

### Test: In Container 2 lesen und antworten

```bash
docker exec -it kn05-volume-2 bash
cd /daten
cat austausch.txt
echo "Antwort von Container 2" >> austausch.txt
cat austausch.txt
exit
```

---

### Test: In Container 1 erneut lesen

```bash

cd /daten
cat austausch.txt
exit
```

## C) Speicher mit Docker Compose

In diesem Teil wird ein Docker-Compose-Setup erstellt. Dabei werden alle drei Speichertypen verwendet: Bind Mount, Named Volume und Tmpfs.

Es werden zwei nginx-Container erstellt. Das Named Volume wird beim ersten Container mit Long Syntax und beim zweiten Container mit Short Syntax eingebunden.

### Ordnerstruktur

```text
kn05/
├── docker-compose.yml
└── mein_skript.sh
```

---

### `docker-compose.yml`

```yaml
services:
  web1:
    image: nginx:latest
    container_name: m347-kn05-web1
    volumes:
      # Named Volume mit Long Syntax
      - type: volume
        source: kn05-named-volume
        target: /data/named-volume

      # Bind Mount mit Long Syntax
      - type: bind
        source: .
        target: /data/bind-mount

      # Tmpfs Mount mit Long Syntax
      - type: tmpfs
        target: /data/tmpfs

  web2:
    image: nginx:latest
    container_name: m347-kn05-web2
    volumes:
      # Named Volume mit Short Syntax
      - kn05-named-volume:/data/named-volume-short

volumes:
  kn05-named-volume:
```

---

### Container mit Docker Compose starten

```bash
docker compose up -d
```

---

### Kontrolle im ersten Container

Im ersten Container müssen alle drei Speichertypen sichtbar sein:

```bash
docker exec -it m347-kn05-web1 bash
mount | grep "/data"
exit
```

## Zusammenfassung

In KN05 wurden drei verschiedene Arten von Docker-Speicher getestet:

| Speichertyp | Erklärung | Beispiel |
|---|---|---|
| Bind Mount | Host-Ordner wird direkt in den Container eingebunden. | `-v "${PWD}:/app"` |
| Named Volume | Docker verwaltet den Speicher selbst. Mehrere Container können ihn nutzen. | `kn05-shared-volume:/daten` |
| Tmpfs | Speicher liegt nur im RAM und ist nach dem Stoppen weg. | `type: tmpfs` |
