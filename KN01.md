Erklärung: Das Image wird automatisch heruntergeladen, da es lokal nicht vorhanden ist. Der Container startet kurz, beendet sich aber sofort wieder, weil Ubuntu kein Hintergrundprozess ist — es gibt keinen laufenden Dienst, der ihn am Leben hält. Im docker ps ist er daher nicht sichtbar, nur bei docker ps -a.
  <p>
<img width="2147" height="435" alt="image" src="https://github.com/user-attachments/assets/7465338e-aed0-4241-9b76-abe85471d3c2" />
Erklärung: Mit -it öffnet sich eine interaktive Shell direkt im Container. Du landest in einem Ubuntu-Terminal und kannst Befehle ausführen. Der Container bleibt so lange aktiv, wie die Shell offen ist. Sobald du exit eingibst, stoppt der Container.

Erklärungen:
docker tag nginx:latest deinbenutzername/m347:nginx
→ Erstellt einen neuen "Alias" für ein bestehendes Image. Das Original bleibt, du gibst dem Image einfach einen neuen Namen/Tag damit Docker weiss, in welches Repository es gepusht werden soll. Ein Tag ist wie eine Versionsbezeichnung oder ein Label für ein Image.
<p>
docker push deinbenutzername/m347:nginx
→ Lädt das lokal getaggte Image in dein Docker Hub Repository hoch. Danach ist es in der Cloud verfügbar.
