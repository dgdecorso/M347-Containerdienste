### 5. Ubuntu – `-d` vs `-it`

**Mit `-d` (Hintergrund):**

![Ubuntu -d](https://github.com/user-attachments/assets/51825042-b9ac-45e1-9a8d-d59106ee68d0)

> Das Image wird automatisch heruntergeladen. Der Container startet kurz und beendet sich sofort wieder, weil Ubuntu keinen Hintergrundprozess hat. Er ist nur bei `docker ps -a` sichtbar, nicht bei `docker ps`.

**Mit `-it` (interaktiv):**

![Ubuntu -it](https://github.com/user-attachments/assets/7465338e-aed0-4241-9b76-abe85471d3c2)

> Mit `-it` öffnet sich eine interaktive Shell direkt im Container. Der Container bleibt aktiv, solange die Shell offen ist. Nach `exit` stoppt er.

![Docker Tag & Push mariadb](https://github.com/user-attachments/assets/bd5c1f88-df88-43f1-a30d-f6acc34f2bc9)

### Befehle & Erklärungen

```bash
docker tag nginx:latest benutzername/m347:nginx
docker push benutzername/m347:nginx

docker tag mariadb:latest benutzername/m347:mariadb
docker push benutzername/m347:mariadb
```

| Befehl | Erklärung |
|---|---|
| `docker tag` | Erstellt einen neuen Alias für ein bestehendes Image. Ein **Tag** ist eine Versionsbezeichnung / Label, das Docker sagt, in welches Repo das Image gehört. |
| `docker push` | Lädt das lokal getaggte Image in das Docker Hub Repository hoch. Danach ist es in der Cloud verfügbar. |
